# Example: variant counts — a resource-only manifest (the agent writes the SQL)

The minimal shape: a manifest that declares **only a resource** (`variants`, a `file_scan` over a local CSV)
and **no operation**. There is no pinned query — the agent does schema discovery and writes the SQL to answer
whatever question it was asked. This is the general path (`runBioQueryFromManifest`): the manifest is the
*data surface*, the SQL is the *program*.

```
variants (file_scan) ──(agent's ad-hoc duckdb.sql)──→ the answer
```

Contrast with [`rare-high-impact/`](../rare-high-impact/), where the safety-critical query is **pinned** as a
declared, tested operation. Here the question is routine, so it stays ad hoc: the agent's read-only SELECT runs
through the same host entry, persists the same `run`/`result`/`receipts`, and pins the SQL digest in provenance.
