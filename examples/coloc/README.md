# Colocalization with SQL and R

The manifest composes two ordinary substrate stages:

1. SQL joins GWAS and eQTL rows, aligns alleles, flips swapped effects, and drops mismatches.
2. `compute.run` sends each harmonized tissue relation to out-of-process R and materializes `coloc.abf` posteriors.

The example is intentionally one locus and three tissues. It demonstrates declared data/compute composition and
receipts, not a production post-GWAS pipeline or biological validation.

```sh
pi-bio-agent query examples/coloc/manifest.json \
  --db :memory: \
  --compute local \
  --init-sql "INSTALL nanoarrow FROM community; LOAD nanoarrow" \
  --sql "SELECT * FROM coloc_result ORDER BY tissue, hypothesis"
```
