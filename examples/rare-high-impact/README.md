# Rare high-impact variant classification

The manifest declares annotated variants, a loss-of-function Sequence Ontology term set, and the repeated operation
`rare_high_impact.report`. The operation assigns every row to one bucket with two explicit safeguards:

- missing allele frequency becomes `no_frequency`, not rare;
- a benign clinical significance excludes the row even when consequence and frequency otherwise pass.

This is a stable operation because the abstention rule is a regression contract. Novel summaries over the same
resources should remain ad-hoc SQL.

```sh
pi-bio-agent run examples/rare-high-impact/manifest.json \
  --db :memory: \
  --operation rare_high_impact.report \
  --run-id rare-high-impact-example
```

Inspect before composing a different query:

```sh
pi-bio-agent query examples/rare-high-impact/manifest.json --db :memory: --sql "DESCRIBE annotated_variants"
pi-bio-agent query examples/rare-high-impact/manifest.json --db :memory: --sql "SUMMARIZE annotated_variants"
```
