# Files-only out-of-process compute

A tool that returns **no table, only files** — the common bioinformatics case (`samtools index` → a `.bai`,
`bcftools` → a `.vcf.gz`, a plot renderer → a `.svg`). `process.compute` supports it with
`resultTable: "artifacts"`:

- **no `inputSql`** → no `in.arrow` is handed in (the tool uses its own inputs / command args), and
- **`resultTable: "artifacts"`** → no `out.arrow` is expected, so the Arrow codec (`nanoarrow`) is **not even
  loaded** — this path is pure files, no IPC.

The child (`./render.sh`) just writes its declared FILE outputs into its work dir (cwd). The resolver captures
each into **CAS** (content-addressed) and — because there is no rectangular value — makes the resource's TABLE
the **artifacts listing itself**: one row per captured output with `name, path, kind, digest, size`. So a
files-only tool is still a first-class, queryable resource, and each output is a CAS handle the agent reads
downstream.

Run it (`sh` only — no R, no network):

```bash

```
